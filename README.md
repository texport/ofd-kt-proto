## Documentation in English

A Kotlin/Java library containing code entities generated from KazakhTelecom OFD (State Revenue Committee of the Republic of Kazakhstan) `.proto` files. 

The current protocol version is **2.0.3** (module `proto-v203`). These proto files implement the data transmission protocol between Cash Registers (KKM) and the OFD.

For other OFD providers, separate repositories and artifacts are intended to avoid namespace conflicts.

### Key Features
- **Pure JVM library**: Compiles to standard Java bytecode, compatible with Java 17+ and Android.
- **Type-safe Kotlin Protobuf bindings**: Exposes native Kotlin classes generated by Google Protobuf Kotlin plugin.
- **Modular design**: Designed to support multiple protocol versions side-by-side in the same project without package collisions.

### Installation

#### 1. Via Maven Central (Recommended)
Add the dependency to your `build.gradle.kts`:

```kotlin
dependencies {
    implementation("kz.kazakhtelecom:ofd-kt-proto-v203:2.0.3")
}
```

#### 2. Via Gradle Composite Build (Local Development)
If you want to compile `ofd-kt-proto` directly from source alongside your main project:

1. Clone the repository next to your main project folder.
2. Include the build in your main project's `settings.gradle.kts`:
   ```kotlin
   includeBuild("../ofd-kt-proto")
   ```
3. Declare the dependency in your main project's `build.gradle.kts`:
   ```kotlin
   dependencies {
       implementation("kz.kazakhtelecom:ofd-kt-proto-v203:2.0.3")
   }
   ```
   Gradle will automatically substitute the binary dependency with your local source code module.

#### 3. Direct JAR integration
Download the compiled JAR from releases and add it to your dependencies:
```kotlin
dependencies {
    implementation(files("libs/ofd-kt-proto-v203-2.0.3.jar"))
}
```

### Local Build Instructions
To build the library JAR locally:
```bash
./gradlew :ofd-kt-proto-v203:build
```
The compiled artifact will be located at `proto-v203/build/libs/ofd-kt-proto-v203-2.0.3.jar`.

### Adding a New Protocol Version
1. Copy the `proto-v203` module folder to a new folder, e.g., `proto-v204`.
2. Update the `version` property in `proto-v204/build.gradle.kts` (e.g., `version = "2.0.4"`).
3. Include the new module in the root `settings.gradle.kts`:
   ```kotlin
   include("proto-v204")
   project(":proto-v204").name = "ofd-kt-proto-v204"
   ```
4. Update the package namespaces in the new module's `.proto` files to prevent package collisions:
   - `option java_package = "kz.kazakhtelecom.proto.v204";`
   - `package kz.kazakhtelecom.proto.v204;`
5. Rebuild the project. You can now use both versions side-by-side:
   ```kotlin
   dependencies {
       implementation("kz.kazakhtelecom:ofd-kt-proto-v203:2.0.3")
       implementation("kz.kazakhtelecom:ofd-kt-proto-v204:2.0.4")
   }
   ```

## Документация на русском языке

Библиотека для использования в Java и Kotlin проектах. Содержит сущности, сгенерированные из `.proto` файлов ОФД Казахтелеком (Комитет государственных доходов РК). 

Текущая версия протокола — **2.0.3** (модуль `proto-v203`). Прото-файлы реализуют протокол передачи данных ККМ в ОФД.

Для других ОФД предполагаются отдельные репозитории и артефакты, чтобы имена пакетов не пересекались.

### Ключевые преимущества
- **Чистая JVM-библиотека**: Компилируется под Java 17+ и полностью совместима с Android.
- **Типизированные Kotlin-биндинги**: Использует нативные Kotlin-классы, сгенерированные плагином Google Protobuf Kotlin.
- **Поддержка параллельных версий**: Разработана так, чтобы несколько версий протокола могли одновременно использоваться в одном проекте без конфликтов пакетов.

### Подключение библиотеки

#### 1. Через Maven Central (Рекомендуемый способ)
Добавьте зависимость в ваш `build.gradle.kts`:

```kotlin
dependencies {
    implementation("kz.kazakhtelecom:ofd-kt-proto-v203:2.0.3")
}
```

#### 2. Через Gradle Composite Build (Для локальной разработки)
Если вы хотите собирать библиотеку напрямую из исходников совместно с основным проектом:

1. Склонируйте репозиторий `ofd-kt-proto` в папку рядом с вашим основным проектом.
2. В файле `settings.gradle.kts` вашего основного проекта добавьте:
   ```kotlin
   includeBuild("../ofd-kt-proto")
   ```
3. В файле `build.gradle.kts` вашего основного проекта укажите зависимость:
   ```kotlin
   dependencies {
       implementation("kz.kazakhtelecom:ofd-kt-proto-v203:2.0.3")
   }
   ```
   Gradle автоматически заменит бинарный артефакт локальным модулем с исходным кодом.

#### 3. Прямое подключение JAR-файла
Скачайте собранный JAR-файл из релизов и добавьте в зависимости:
```kotlin
dependencies {
    implementation(files("libs/ofd-kt-proto-v203-2.0.3.jar"))
}
```

### Сборка библиотеки локально
Для сборки JAR-архива локально запустите команду:
```bash
./gradlew :ofd-kt-proto-v203:build
```
Собранный файл будет находиться по пути: `proto-v203/build/libs/ofd-kt-proto-v203-2.0.3.jar`.

### Добавление новой версии протокола
1. Скопируйте папку модуля `proto-v203` в новую, например `proto-v204`.
2. В файле `proto-v204/build.gradle.kts` обновите `version` (например, `version = "2.0.4"`).
3. Добавьте модуль в корневой `settings.gradle.kts`:
   ```kotlin
   include("proto-v204")
   project(":proto-v204").name = "ofd-kt-proto-v204"
   ```
4. Обновите пространства имен в `.proto` файлах нового модуля, чтобы избежать пересечения пакетов:
   - `option java_package = "kz.kazakhtelecom.proto.v204";`
   - `package kz.kazakhtelecom.proto.v204;`
5. Пересоберите проект. Теперь вы можете подключать обе версии одновременно:
   ```kotlin
   dependencies {
       implementation("kz.kazakhtelecom:ofd-kt-proto-v203:2.0.3")
       implementation("kz.kazakhtelecom:ofd-kt-proto-v204:2.0.4")
   }
   ```
